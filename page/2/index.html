python过滤特定词练习

题目：

敏感词文本文件 filtered_words.txt，里面的内容为以下内容，当用户输入敏感词语时，则打印出 Freedom，否则打印出 Human Rights。

我的思路是这样的，就是把这个文件读进一个数组，再判断raw_input输入的在不在这个数组。

这里说明下，这里用的读取文件的方式不好，后面我会写一个关于怎么去读文件的文章吧，大家尽量不要用readlines

这里有个坑，就是filtered_words.txt文件在中文的时候就一直不正确，试了各种转码（转码这里也有一些问题，不知道具体的方式），debug发现正常存到数组的中文是转成GBK编码了，正常的输入其实也GBK，但就是不一样，所以愁了半天，后来发现这个txt文件在pycharm打开是乱码，今天就去查了下windows记事本带的编码格式，有以下三种：

  1. ANSI（默认的）
  2. Unicode（带有BOM格式的UTF-16）
  3. UTF-8带BOM

个人猜的就是这个带Bom的问题，如果文本中所有字符都在 ASCII 范围内，那么其实，记事本保存的所谓的「ANSI」文件，和 ASCII 或无 BOM 的 UTF-8 是一样的，所以正常英文是没有问题的啊，正好在ASCII范围内，但中文就出现在保存上了。
正常的保存的方式就是要保存UTF-8不带BOM，一般常用的noetpad++，就可以设置为这个格式，也可以用各种大型的IDE，爱中文，要远离记事本编辑啊。

自动打包功能探索

问题

公司现在有几个版本的apk，打包全是在eclipse上签名打包，正式版本、测试版本都是这样，昨天花了半的时间去了解具体相关的打包流程，结果发现公司项目编码是utf-8+BOM格式，eclipse能自动转化成utf-8，所以之前一直没有问题，所以用正常的打包的方式就行不通了啊，等项迁移到AndroidStudio就再说吧，说下我之前公司的打包流程：
整体是在一个jenkins+ubuntu+svn+shell，在jenkins上写好对应的shell命令，shell命令主要工作就是检查各个工具是否能正常使用，能否从SVN上checkout原码、还有混淆等基本打包流程的代码。开发切分支开发的就是简单的在服务器上复制一个目录，所以针对不同版本的打包就只用复制jenkins的一个项目，再把shell里面的checkout的路径修改就下ok了；所以测试只需要在jenKins点击下开始构建的按钮就可以完成打包流程，打包好的文件也会放到ftp上，自己再定个命名规则，就可以写成比较麻烦的打包流程了。

流程

先放出一个详细的打包流程的东西：

正常的打包一个apk是比较麻烦的事情，正常的有以下几步要做的：

  1. 生成R文件（aapt）
  2. Java代码编译成class文件（javac）
  3. class文件生成dex文件(dx)
  4. 打包资源（aapt）
  5. 生成apk（apkbuilder）
  6. 创建密匙（keytool）
  7. 签名apk（jarsigner）

括号里面写的就是要使用的工具，在SDK里面都有，正常的打包是这样的，但实际中还有一些混淆之类的，主要为了安全做的啦，下面把上面这些工具怎么使用的方法粘出来吧，是唐老板写的，我找时间也会玩下（其实现在都用ant、maven、gradle了，很方便的）

下面用Ubuntu命令行来打包，流程如下：

  1. 生成R文件，命令如下：
    aapt package -f -m -J $BUILD_SOURCE_PATH/$ANDROID_GEN \
    -A $BUILD_SOURCE_PATH/$PACKAGE_RESOURCE_FILE \
    -S $BUILD_SOURCE_PATH/$ANDROID_RESOURCE_DIRECTORY \
    -I $BUILD_API_LEVEL_JAR \
    -M $BUILD_SOURCE_PATH/$MANIFEST_FILE

-f 如果编译生成的文件已经存在，强制覆盖。
-m 使生成的包的目录存放在-J参数指定的目录
-J 指定生成的R.java 的输出目录路径
-S 指定res文件夹的路径
-I 指定某个版本平台的android.jar文件的路径
-A 指定assert文件夹的路径

2.将Java代码编译成class文件，命令行如下：
javac -encoding UTF-8 \
-target 1.7 \
-bootclasspath $BUILD_API_LEVEL_JAR \
-d $BUILD_SOURCE_PATH/$ANDROID_BIN/$ANDROID_BIN_CLASSES \
$BUILD_SOURCE_PATH/$ANDROID_GEN/com/clarck/httpclientnew/R.java \
$BUILD_SOURCE_PATH/src/com/clarck/httpclientnew/*.java

-encoding <编码> 指定编码格式
-target <版本> 生成特定 VM 版本的类文件
-bootclasspath <路径> 覆盖引导类文件的位置
-d <目录> 指定存放生成的类文件的位置
-sourcepath <路径> 指定查找输入源文件的位置

  1. class文件生成dex文件，命令行如下：
    dx —dex —output=$BUILD_SOURCE_PATH/$ANDROID_BIN/$ANDROID_CLASSES_DEX \
    $BUILD_SOURCE_PATH/$ANDROID_BIN/$ANDROID_BIN_CLASSES

—output=<要生成的classes.dex路径> <要处理的class文件的路径>

  1. 打包资源,命令行如下：
    aapt package -f -M $BUILD_SOURCE_PATH/$MANIFEST_FILE \
    -S $BUILD_SOURCE_PATH/$ANDROID_RESOURCE_DIRECTORY \
    -A $BUILD_SOURCE_PATH/$PACKAGE_RESOURCE_FILE \
    -I $BUILD_API_LEVEL_JAR \
    -F $BUILD_SOURCE_PATH/$ANDROID_BIN/byreadreader

-f 如果编译生成的文件已经存在，强制覆盖
-M 指定AndroidManifest.xml的路径
-S 指定res文件夹路径
-I 指定某个版本平台的android.jar的路径
-F 指定输出文件完整路径

  1. 生成apk，命令行如下：
    apkbuilder $BUILD_APK_PATH/$BUILD_APK_NAME \
    -v -u -z $BUILD_SOURCE_PATH/$ANDROID_BIN/byreadreader \
    -f $BUILD_SOURCE_PATH/$ANDROID_BIN/$ANDROID_CLASSES_DEX \
    -rf $BUILD_SOURCE_PATH/$ANDROID_SRC

-v Verbose 显示过程信息
-u 创建一个无签名的包
-z 指定apk资源路径
-f 指定dex文件路径
-rf 指定源码路径

  1. 创建密匙，命令行如下：
    keytool -genkey -alias release -keyalg RSA -validity 20000 -keystore release.keystore

-genkey 在用户主目录中创建一个默认文件”.keystore”,还会产生一个mykey的别名，mykey中包含用户的公钥、私钥和证书
-alias 产生别名
-keyalg 指定密钥的算法
-validity 指定创建的证书有效期多少天
-keystore 指定密钥库的名称(产生的各类信息将不在.keystore文件中)

  1. 签名apk，命令行如下：
    jarsigner -verbose -keystore release.keystore -storepass !@#$%^ -keypass !@#$%^ -signedjar $BUILD_APK_PATH/$BUILD_RELEASE_APK_NAME $BUILD_APK_PATH/$BUILD_APK_NAME release

-verbose 签名/验证时输出详细信息
-keystore 密钥库位置
-storepass 用于密钥库完整性的口令
-keypass 专用密钥的口令（如果不同）
-signedjar 已签名的 JAR 文件的名称 （第一个apk是签名之后的文件， 第二个apk是需要签名的文件）

注：关于混淆代码，只需要在project.properties中指定用來混淆的.pro文件即可。
以上命令可以综合使用一个shell脚本来执行，具体demo详见：https://github.com/clarck/CommandAutoBuild

最后一个是重点啦，下次就把前面说到的三个打包工具拿出来讲怎么打包成功。等我哈。。

　　这一年为了能让自己好好的看书，买了一个Kindle（没有收藏纸质书的习惯），期间断断续续看了6、7本书吧，收获还是有不少的，至少知道思考不能停，但有个比思考更重要的，就是做。有时候就是要简单的去做，不用去想why，用心去做，不因为心情去影响做事的用心，包括工作和感情上的。还有就是现在心又大了好多，我也会生气，生气了就缓几分钟再去对这些事情处理，这样看起来有点软了，但其实是事少了，人生那么短，为那些事情生气那么久干嘛，对吧。最近看了有一个关于整理的书，也明白一个朋友说过一句，怎么去提升自己的生活品质，就一个方法“经常扔东西”（这里感谢那位朋友，虽然半年才算明白），其实就是尽量少买东西，还有处理现有的东西，选些有品质的生活。生活是用来过的，不是用来将就的，这个想法也是最近才搞明白的，会去提升自己日常生活的品质。跑步是一个最无聊的运动，其实这个也没有什么错，只是单纯的跑步，肯定是无聊啊，但有个好玩的，就是跑步中去思考一些问题，感觉效率好高，跑完后休息的效率也很高，所以这个跑步会一直跑下去，不用天天跑，累了，不爽了，开心了，随心去跑。跑马也让自己明白有些东西是更多的加油坚持，少许的学会放弃。更多中性的想法去看待问题，看待人生，错的不是人，是人的思想，学会换个角度思考，换个方式过自己的生活，也就够了，所以2016年的目标不多，就是把自己的思想落地了，再出来一些也许、可能比较好的思想方式呢，目标还是一个字：开心。

工作

　　这一年换了一份工作，换工作呢，是正常的，上面也说过原因，不是工作不好，只是不适合自己而已，只是想在自己先阶段做自己想做的事情，好好的把工作的事情做好，毕竟男人嘛，事业的好坏是最直接的说明了，今年各种互联网资本寒冬，但感觉对自己的影响不大，不创业，不投资，还是做好自己的老实人，这个远的，现在不想了；今年的工作换了一个方向，但工作性质的问题，新方向做的不多，其中发现一个问题，自己有时候是笨的可以，哈哈，实话啦，目前只能用时间去堆这个笨的差距咯，还有一个问题就是实践的有点少了，不要眼高手低的去看问题，不去做就不知道有多少坑等着自己。现在的自己也不小了，自己的重心在慢慢转向工作中，这个去年就说过啦，只是还“年轻”，玩的自然会多点啦，所以明年的计划，新的工作方式继续做好，用心去做自己的工作，去实践一些眼高手低的事情，还有就是完成自己的一个app，就是玩玩的，有点必须的技能基础，还有每个月给自己整理一个技术学习总结文章，两个月至少一个工作、生活总结的要求，目前就这些吧，别的看情况再慢慢定，第一次给自己目标，好有压力的感觉，先定下来吧，2016看自己的啦，加油吧。

旅行

　　这个呢，想想还是单独拿出来吧，其实上面列出来的基本都是出游的，因为别的事情自己也不会记太多，所以给人的感觉就是我各种出游，其实是我基本只发出游的朋友圈，也只是利用周末出去而已，工作必须还是好好的做着呢；16年的想法把15年没有实现的那个出国走了，出大陆走走也行啊（目标不大），今年游的很多地方都是走马观花的感觉，看起来走了不少地方，但其实也没有玩多少，所以改变玩的形式，少玩，深玩，重点转向工作，尝试自己去安排所有的行程，学会自己去玩，去做游记，剩下的好像也没有什么好写的了，目前就这些吧

结束

　　上面写的自己看的都乱，没有想去讨好谁，主要也是为了一个记录，将就的看着，加油的过着。也祝福看到这里的朋友在2016年工作顺利、身体健康。

15年07、08、09、10这四个月的总结

生活

之前定的目标，现在完成的好像也就一个跑步吧，哈哈，而且是那种上瘾的，其实事后想想，不全是上瘾吧，有部分是炫耀吧，看下跑步比赛的记录，训练的就不算了，就有这么多了呢。

  • 8月一个半马
  • 9月一个全马、一个半马
  • 10月一个半马、一个越野半马
  • 11月一个全马

想想都感觉有点过了，不过是下决心跑下去了，也是因为这些看到了一些跑者的态度和一些跑完细成狗，但还是很高兴的经历；膝盖已经疼的不行了，停下来是真的不想跑了，但跑起来又没有感觉了，路边加油的陌生人其实就一道美丽的风景线，我只是一直路过看风景的，其实这几个月没有怎么写技术性的文章也算是有原因的吧（不是逃避，是Summary），之前看到一遍文章写的，有个地方写到，生活没有必要让自己那么累，有时间学会放松下自己，虽然人生是需要各种加油的，要前进，停下来看看自己的过去，享受自己之前努力的成果，放松的去享受这个世界，所以就有了这几个月基本不写什么技术的东西了，好了，现在开始，继续学习吧。

工作

7月份的时候，和猎豹谈说了离职的申请，原因是想改变，也感谢那个她（一个又爱又恨的“朋友”），当时想了很久了，但那个不是自己的终点，所以换着吧，做了刚刚好两个月吧，前期有点压力，但也完成一个简单的框架，也能完成基本的测试了，后期再慢慢调试吧，这个也算是一个学习吧；还有一个比较纠结的问题，就是学习的心啊，JAVA看了挺久的了，结果也看不了多少，主要是单看书，自己感觉是看不懂的，感觉自己笨笨的，好吧，有超级笨，现在的想法不是多去尝试吧，怎么说自己做测试也有三年了，前一年半算是混过去的，所以现在就是要补回来，最近还有看到一个关于做事情用心的事情，这个呢，想想自己也好久没有特别用心去做一些事情，不爱感情的、工作的，还是什么的，其实自己有能力做好，但自己还是有点懒，其实这个用心是让自己在做一些事情上会有特别明显的收益，也明白有些事情你做过，和没有做过是一个等级，做过和细心去做又是一个等级，也只有用心的去做每一件事才会让这件事对自己有更大的收益，其实上面这些自己也是懂，但自己真心去明白和做过感觉是一个进步吧。

后期

想想也快过年了，今年收获的不是很多，不对，应该是有很多 ，都是去玩的啦，比如上面的马拉松，过程是感觉超级累，但跑完那种感觉还是棒棒的，哈哈，不过天气冷了，自己也懒了不少，最近拖延症犯的有点严重了，比如这个日志吧，是12月初才写完的，还有几个大视频要去剪辑，还有好多东西要看，想想就各种忙，但现实自己看自己又是各种无聊，和一个朋友好聚好散，也许是自己的不会珍惜，也可以说成不够用心吧，也许也是她的不用心，呵呵，不管怎么样吧，都这样了，就过去吧，就像我网名那样，幸福有时候会迟到，但它从不缺席，感谢这几个月的生活，有高有低，终回平静，8月的换工作，9月的人生第一个全马，10月的拉萨，都是自己以前想都没有想过的事情，也许这个对于别人来说，很容易，但我想说的是，这些对于我来说，够了，够开心的了，能在25岁完成这些，我很开心，但还没有满足，之前的想法是有个人能好好的陪自己去体验这些不一样的生活，但目前看来，好像是没有了，没事，自己去体验也不是挺好的吗，买了三角架，带了遥控器，想自己自拍就怎么自拍，其实这些就想到了素瑶，一个认识也就8个多月的妹子，感觉一下子就成了”闺蜜”了，和她在一起少了很多担心的东西，拉萨也是她拉我去的，玩的还行，也感觉她的帮忙拍照，怎么看怎么感觉我们是发小的感觉，哈哈，不过还好，在一起开心就行。
也快过年了，明年好好的给自己一个年计划，今年的本来后半年的计划全泡汤了，没有几个做好的，说写的技术类的，感觉以前写的有点Low，不对，是太Low了，自己就满足自己那点小成就可不行啊，慢慢的有点深度，最近早上天天看懂懂的日志，感觉虽然有些是不相信的，但他写的东西，可以让自己好好的去想想之前见过的，体验过的，换个角度想问题，会有不少你自己不知道的效果。上周还做过一个演讲，公司内部的，题目是【折腾的意义】说的都是自己这些日子的折腾的东西，其实之前想的很好，但没有太用心去做，应该在讲之前自己多试，想上台就说的很好，这个不太可能吧，一个真正说的好的不是一上台就牛逼的，也是在台下练了我自己也想不到的很多遍就对了，所以呢，用心去做一件最直接最有效的方式就是去多做，但你做过了你才知道别人说的经验是什么，几年后回来看看这遍自己写的博客会怎么样，呵呵，那个是以后的事，这里只是记录自己一路过来的地方。
