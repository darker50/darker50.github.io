安装前准备

这里有个比较麻烦的问题，因为我们的App加密使用的是256位加密，一般的jdk、jre是运行不了，这里要替换两个文件，先看自己JAVA的版本，就去下对应的JCE，替换jdk1.8.0_60\jre\lib\security和jre1.8.0_60\lib\security目录下
Java Cryptography Extension (JCE) Unlimited Strength Jurisdiction Policy Files 6

Java Cryptography Extension (JCE) Unlimited Strength Jurisdiction Policy Files 7 Download

Java Cryptography Extension (JCE) Unlimited Strength Jurisdiction Policy Files 8 Download

解密说明

这里踩坑的原因，AES默认是只支持128加密的（这个是美国自己规定），替换后就支持256位密钥了，因为正常的运行在IDE运行java工程调用的是jdk里面的jce文件，打包出来运行的就jar包调用的就是jre里面的jce文件，开始不了解这个原理就只修改对应的jdk的jce文件，结果打包出来后就一直报那个已经解决的问题，就是要替换对应的jce文件的问题，后来替换了就成功了，设置完之后用正常的方式Import打好的jar包，就可以调用jar做一个解密接口了。
