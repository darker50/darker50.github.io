AES中Key的超过128的解决方式

介绍

维基百科说明：
AES是Advanced Encryption Standard缩写，高级加密标准，在密码学中又称Rijindael加密法，是美国联邦政府采用的一种区块加密标准，是替代之前的DES的，属于对称密钥加密中最流行的算法之一。
剩下的东西有点专业，从我自己理解的角度就是说，一个字段中，截取一部分做密钥，剩下的一部分做解密的数据，剩下的不用管了，用一些自带的方法就能解决。
这种是128位换算方式

128bits == 16Bytes == 16 Chars

原因

java.security.InvalidKeyException: Illegal key size or default parameters

使用AES加密、解密过程中会有这个问题，原因呢是AES加密正常的要求密钥是为128位，超过128位时，就报错误咯，这个限制其实是美国对软件出口的控制，谁叫AES是美国发明的呢，但我们不是美国人，所以要解决这个问题

解决方法

就是替换jdk中下的security的local_policy.jar和US_export_policy.jar两个包
对应版本的下载地址：

关于python一些编码的问题

常见的编码类型

先说明下常用的单位换算方式：8比特（bit）=一字节（byte），大家都懂的，不同的编码就是用不同的字节来表示对应的文字，比如英文用ACSII就可以了，中文可以用Unicode、utf-8等，下面说常见的编码格式

  • ASCII:最早期的编码格式，只支持大小写字母、数字加一些符号—一个字节
  • GB2312:是中国自己定制（之后还有类似的日文的Shift_JIS、Euc-kr）—两个字节
  • Unicode:把上面所有的编码统一上面说到的所有编码—常用两个字节，也有用四个字节的
  • utf-8:把一个Unicode字符根据不同的数字大小编码成1-6字节，英文：1字节，汉字：3字节，有些特殊的会用到6字节

官方说明

open(filename, access_mode=’r’, buffering=-1)

filename：文件的相对、绝对的路径
access_mode：打开文件的模式，’r’,’w’,’a’，’b’ 来打开，分别代表，读取，写入，追加，打开二进制文件，其中‘w’模式打开的文件若存在，则先清空，再写入内容
buffering:缓冲方式，0：不缓冲，1:只缓冲一行数据，-1：代表使用给定的值作为缓冲区大小。不给定此参数或者参数为负数都表示使用系统默认的缓冲机制。

setdefault(key[, default])
If key is in the dictionary, return its value. If not, insert key with a value of default and return default. default defaults to None.
